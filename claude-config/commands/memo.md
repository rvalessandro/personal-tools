# Memo

You are helping draft a team memo following an established format.

## Memo Format

```markdown
# [Title - clear and specific]

## The Problem

**[One-liner stating the core issue]**

Currently:
- [Bullet points describing the current state]
- [Pain points and symptoms]

## The Solution

**[One-liner describing the solution]**

[Explanation of the approach with tables/diagrams if helpful]

## [Detailed Sections]

[Break down the solution into actionable sections]
[Use tables, bullet points, and examples]
[Include specific processes, rules, or rituals]

## What This Means For You

**[Role 1]**:
- [Specific actions for this role]

**[Role 2]**:
- [Specific actions for this role]

**Everyone**:
- [Universal expectations]

---

**Bottom line**: [One-liner summary of the key takeaway]
```

## Tone & Style

- Direct and clear — no corporate fluff
- Bilingual OK if appropriate for the team
- Use tables for comparisons
- Use bullet points for lists
- Include specific examples
- Frame changes positively (enable, not punish)
- Connect to existing memos/processes when relevant

## Process

1. Ask the user what the memo is about (or use provided arguments)
2. Ask clarifying questions if needed:
   - Who is the audience?
   - What's the current pain point?
   - What's the proposed solution?
3. Draft the memo following the format
4. Show draft and ask for approval/changes
5. Ask where to post it (Linear, Notion, Slack, etc.) — don't assume
6. Post only when user confirms destination

## Rules

- Keep "The Problem" and "Bottom line" to one line each
- Be specific — no vague language
- Include actionable guidance for each role
- Use real examples when possible
- Don't post until user approves AND specifies destination

Start by asking: "What do you need to communicate to the team?"
